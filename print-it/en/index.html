Web development tools

Let's hold our breath and dive in the deep and dark ocean lying below Print-it. Don't forget your headlamp, knowledge is at the end of the journey.

Atom

Atom is a text editor for mac-os, linux
and Windows, built to write source code.
We used Atom to write the HTML structure, CSS style sheets and Javascript Find out more about those acronyms on page 10. code composing Print-it, shaping it bit by bit.

Jekyll

Jekyll is a static site generator, build by Tom Preston-Werner, written in the programming language Ruby. As friendly as it gets, Jekyll is friendly, but what about Mr Hyde ? :o it is one of the most used static site generator out there.
Jekyll was in charge of synchronizing the different parts of the code (HTML/CSS/JS) of Print-it.

GitHub

Github is a web platform for code hosting and management, based on the version control software Git. Version control is a starting point for collaboration workflows often associated with open source projects.
Github is the host of Print-it's source code, it allows us to regroup and synchronize all the modifications made on the software.
08

Siteleaf

Developed by Oak Studio, Siteleaf is the Ideal sidekick of Jekyll. It connects to Github and hosts the site on a specified URL. It comes with a CMS, providing an easy way to update the content of the site.
Siteleaf is synchronizing all the modifications we are making to Print-it via the code and the CMS, and displays the result on our URL: https://objetpapier.fr/print-it/

Chrome

Chrome is a web browser developed by Google This small family business begged us to advertise their service, we couldn't refuse :/ , it's a software built to access URLs and display the content of the World Wide Web. Fun fact, the french word for browser is "navigateur", which was inspired by
Netscape Navigator, a major web browser from the 1990's. Nowadays, the most used browsers are Google Chrome, Mozilla Firefox,
Microsoft Edge and Apple Safari.
Thanks to Chrome you can see the web interface of Print-it's home page with your own eyes. We highly recommend the use of Chrome for the print process, because it's the one and only browser supporting CSS print like a boss!

From Web to Print

HTML

HyperText Markup Language

HTML is a data description language, structured with tags, built to organize the content of a web page. It's the backbone of internet content, and one of the three pillars allowing the web to exist with HTTP and URL.

CSS

Cascading Style Sheets

The lowlands of the Internet are full of symbols hostile to vowels, who knows where they are hidden? Cascading style sheets is the language used to create the layout and graphics of web content. It's responsible for the appearance of a web page.

CSS Print

CSS Print is the group of instructions written in CSS, applying to the content when it is printed. This way it is possible to work on print layout design using a web development environment, creating a bridge between physical and digital worlds for our content.

PDF

Portable Document Format

Hold on, the fun is coming your way! Best known under the acronym than the actual name, a PDF is a file using a descriptive language to interpret page layout. It was introduced by Adobe Systems in 1992 and became an ISO standard in 2008. It's secret weapon is to be able to preserve the layout exactly the way the author created it, no matter what the software, operating system or computer used to display or print it.
10

Glossary

Source code The source code of a computer program is a text containing instructions written in one or multiple programming languages, it's interpreted by the machine, which executes those instructions.
CMS (Content Management System) A kind of program allowing the modification of a website's content, through a graphical user interface.
Static site A site displaying the same page for every user accessing it on the internet. In opposition to a dynamic site which can display specific content for every user.
Programming language A logic based system made to write source code. Like natural language, each programming language uses an alphabet, a semantic, a vocabulary and a specific syntax.
URL Designates the web address allowing a browser to locate any resource online (file, image, web page, etc.). Formalized by Tim Berners-Lee in 1989, the protocol behind URLs is one of the inventions which made the World Wide Web possible.
HTTP (HyperText Transfer Protocol) This is the communication method used by devices accessing the internet, like a server or a browser. HTTPS is the secure version of the HTTP protocol.
Open source In opposition to the proprietary software philosophy, the open source is a development method based on collaborative work and source code sharing.
Version control Consists in maintaining the modifications made to one or multiple files by one or multiple people. It's mainly used in software design, in order to maintain and track the evolution of the source code.

What is GeoCities?
In november 1994, the company Beverly Hills Internet offers a fifteen mega octets hosting space for free on the web. This paltry amount of storage would be saturated with a few photos today but it was a lush promise at the time, attracting thousands of users around the globe in the first few months of the company's existence. The founders changed the name to GeoCities, and from What a story! a simple hosting service, it quickly evolved into a platform we could consider as an ancestor of blog platforms, where you could customize pretty much everything. If the first iterations of the service asked for a few developing skills to be fully mastered, it changed overtime to be more accessible. It's user experience was getting better and better, attracting more and more users, so you could create your own internet space even without having a programming background. At the end of the nineties, Internet is still in it's early days, and attracting users is easier if they can rely on something already known. This goal in mind the creators of GeoCities introduced a "skeuomorphic" dimension in their product, making you move in your cyberspace like you move into a new flat. Like in real life you chose a city where to live, and a neighborhood where you were given a number, creating your address in this digital world, where you can shape your dream space exactly how you want and share it with the planet. GeoCities is a free minded place promoting expression, where teenage blogs are adjoining war veteran writing their Memoirs and Japanese pop-stars fan-sites. That's what we call convergence among social struggles
This huge variety of topics is coming to life through an aesthetic of gifs, fonts and word-arts in "Web Safe Colors" (216 colors). Print-it is reviving this internet of the past, giving you the keys to create a GeoCities page worthy of it's name.
The essentials of a GeoCities page
  • Mosaic backgrounds 1000 pixels wide background images didn’t Do well with 56k connections. In 1994 backgrounds are made with small images repeated across the page, a bandwidth love story.

  • "Microbars" or "Stamps" It's those small images with a specific size you could find across headers and sidebar with a purpose of claiming an opinion about which web browser to use; or simply your devotion to cat fluff.
  • Animated GIFs The graphics interchange format is known for being able to display image sequences like films in a lightweight format. It was already a killer well before snapchat, mostly on GeoCities.
  • "Under Construction" section A GeoCities page is a constantly evolving work of art, you should never forget to remind your user with appropriated graphics. So he does not forget to come back for your next update. The best update.
  • Multiple typefaces, in image formats The nineties' internet was a very limited place for fonts, allowing the usage of a few system font. Your graphic design talent could only be showcased through pixel based images.
  • "Hotlinks" Hotlinks were links using resources from another website, mainly images. Not a good practice since bandwidth was billed on usage. The word was often misused to define simple page links on GeoCities.
  • "Frames" Defining a subdivision of a webpage containing another html page with it's own scroll. Allowing more complex designs, often resulting in a catastrophic user experience of the website.
  • Fixed layout Computer monitors didn't have a variety of size like devices nowadays. The web was much easier to design because of the size norm: 800 pixels wide and 600 pixels high.
References made to GeoCities
A digital metropolis once overcrowded and constantly developing, the neighborhoods of GeoCities were deserted. It's people fled to a new Eldorado focused on perfected interactions between users: social networks. Monitored browsing and marketed content is now the norm of the internet, far from the freedom you could find at the time. The GeoCities experience regained of it's attractiveness for a lot 90's users, many worked on projects in reference to this space where free expression was the main goal, we selected some of them for you to discover on the next page.

Cameron's world
The most convincing love letter to GeoCities is undoubtably Cameron's World, a website by Cameron Askin. A gold mine of content and visual straight out of the 1996's blogosphere, it a journey through multiple neighborhoods, each one more vivid than the one before. It's a striking experience, maybe more authentic than the original?

Deleted City
Fifteen years of service and 650 Go of archives, GeoCities is a hell lot of internet not the most friendly to exploration. Richard Vijgen is a great help on the subject with this interactive map of the archive. Deleted city is also the best way to visualize the actual size of the amount of websites created.

GeoCities-izer
Now that you read this article, the one and only this you're dreaming of: that the whole internet looks like GeoCities. GeoCities-Izer is making this dream come true, paste the URL of your favorite website to transform it in a "GeoCities ready version", so you won't have to deal with usability and all this nonsense.

NeoCities
The aesthetic of GeoCities websites can look outdated for a lot of us, the trend is coming back. Thousand of users created modern versions of GeoCities websites, keeping the visual style and iconic components, adding all the pop culture created on the internet for the last ten years. NeoCities is the place where you can find the work of all those NetScape fans.

All good things come to an end
The service was bought by Yahoo and slowly abandoned by it's users, to finally be closed in 2009. The Archive Team saved the 650 gigs archive before the closing, it can be found on multiple websites. An investigation led by Corentin Moussard The Japanese version of GeoCities lasted for a decade more, she just closed in march 2019, perfect timing to celebrate the GeoCities culture.
